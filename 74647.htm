Sauvegarder et restaurer les bases de données

Pour sauvegarder une base de données, suivez l'une de ces méthodes :

  • Exporter un dump de la base de données au format SQL. Cela vous permet de créer et de télécharger une base de données dans un fichier unique. Un fichier unique peut être restauré dans la même base de données ou dans une autre base de données. Pour en savoir plus, consultez la section Exporter et importer les dumps de bases de données.
  • Sauvegarder un abonnement complet. Avec cette méthode, vous pouvez créer un fichier de sauvegarde dans le stockage sélectionné (sur le serveur Plesk ou sur un serveur FTP distant). Quand vous restaurez la sauvegarde, vous pouvez sélectionner la base de données à restaurer. Consultez la section Sauvegarder et restaurer les sites Web pour en savoir plus.

Pour restaurer une base de données depuis un fichier archive de sauvegarde de l'abonnement ou depuis un fichier dump SQL, procédez comme suit :

  • Importer un dump de base de données. Cela vous permet de restaurer les données d'un fichier au format SQL. Vous pouvez restaurer les dumps individuels dans toute base de données du même type que le dump (par exemple : MySQL). Pour en savoir plus, consultez la section Exporter et importer les dumps de bases de données.
  • Restaurer une base de données d'une sauvegarde d'un abonnement complet. Quand vous restaurez la sauvegarde, vous pouvez sélectionner la base de données individuelle à restaurer. Consultez la section Sauvegarder et restaurer les sites Web pour en savoir plus.
