Droits disponibles pour le site Web

Ci-dessous, vous trouverez la liste des droits pour toutes les opérations que vous êtes autorisé à effectuer dans votre session Plesk (ces droits se trouvent dans la section Compte > onglet Droits) :

  • Gestion de la zone DNS. Gérez les enregistrements de ressources dans les zones DNS des sites Web. DNS signifie "Système de noms de domaines". Il s'agit d'un service qui permet aux navigateurs Web de localiser les sites Web par les noms de domaine.

    Pour en savoir plus sur la configuration des paramètres DNS de vos sites Web, reportez-vous à la section (Avancé) Configurer le DNS pour un domaine.

  • Gestion des paramètres d'hébergement. Gérez les paramètres d'hébergement Web, comme les paramètres de serveur Web et la prise en charge des langages de script.
  • Gestion des paramètres PHP communs. Ajustez les paramètres PHP communs (comme safe_mode, open_basedir, etc.) pour vos sites Web.
  • Gestion de l'accès au serveur via SSH et Gestion de l'accès au serveur via la Connexion au Bureau à distance. Connectez-vous en toute sécurité au serveur pour gérer les fichiers et envoyer le contenu Web via Secure Shell (Linux) ou une connexion Bureau à distance (Windows).
  • Gestion du FTP Anonyme. Configurez un dossier sur un serveur qui doit être accessible aux utilisateurs Internet via le protocole FTP. Ce dossier peut par exemple avoir une adresse du type ftp://telechargements.exemple.com. Cette fonction est appelée FTP anonyme car les utilisateurs n'ont pas à indiquer un nom d'utilisateur et un mot de passe pour parcourir, télécharger et envoyer des fichiers.

    Pour en savoir plus sur la configuration de dossier FTP sans accès limité, reportez-vous à la section Configurer l'accès FTP anonyme.

  • Gestion du planificateur. Planifiez l'exécution des programmes ou scripts dans le système d'exploitation du serveur.

    Pour en savoir plus sur la planification des tâches, reportez-vous au chapitre Planifier les tâches.

  • Gestion du filtre anti-spam. Configurez des paramètres personnalisés de protection contre les mails commerciaux indésirables, également appelés "spams".

    Pour en savoir plus sur la configuration du filtre anti-spam, consultez la section Protéger les boîtes mail contre les spams.

  • Gestion de l'antivirus. Configurez des paramètres personnalisés pour la protection contre les virus et autres logiciels malveillants diffusés par mail.

    Pour en savoir plus sur la configuration de la protection antivirus, consultez la section Protéger les boîtes mail contre les virus.

  • Sauvegarde et restauration des données d'abonnements à l'aide du stockage du serveur. Utilisez l'outil Gestionnaire de sauvegardes de Plesk pour sauvegarder et restaurer les données de l'abonnement : sites Web, comptes mail, bases de données, logs et statistiques, pages d'erreur personnalisées, listes de diffusion, certificats SSL/TLS et informations DNS. Vous pouvez ainsi conserver vos fichiers de sauvegarde sur le même serveur que Plesk.

    Pour en savoir plus sur la sauvegarde et la restauration des données, consultez le chapitre (Avancé) Sauvegarder et restaurer les sites Web.

  • Sauvegarde et restauration des données d'abonnements à l'aide d'un stockage FTP. Utilisez l'outil Gestionnaire de sauvegardes de Plesk pour sauvegarder et restaurer les données de l'abonnement : sites Web, comptes mail, bases de données, logs et statistiques, pages d'erreur personnalisées, listes de diffusion, certificats SSL/TLS et informations DNS. Vous pouvez ainsi conserver vos fichiers de sauvegarde dans un répertoire FTP sur le serveur de votre choix.
  • Sauvegarde et restauration des données de comptes à l'aide du stockage du serveur. Utilisez l'outil Gestionnaire de sauvegardes de Plesk pour sauvegarder et restaurer vos données de comptes et d'abonnements et utilisez le stockage sur le serveur Plesk pour conserver les sauvegardes. Remarque : ce type de sauvegarde n'est pas disponible par défaut afin d'éviter toute confusion avec la sauvegarde de l'abonnement (sans les données du compte).
  • Sauvegarde et restauration des données de comptes à l'aide d'un stockage FTP. Utilisez l'outil Gestionnaire de sauvegardes de Plesk pour sauvegarder et restaurer vos données de comptes et d'abonnements et utilisez des serveurs FTP externes pour conserver les sauvegardes. Remarque : ce type de sauvegarde n'est pas disponible par défaut afin d'éviter toute confusion avec la sauvegarde de l'abonnement (sans les données du compte).
  • Gestion des statistiques Web. Configurez les préférences personnalisées des rapports statistiques des visiteurs.

    Pour en savoir plus sur la configuration et l'affichage des statistiques sur les visiteurs de sites Web, consultez la section Voir les statistiques.

  • Gestion de la rotation des logs. Configurez les préférences personnalisées pour le recyclage (rotation) des logs de serveur Web. Le serveur Web enregistre les informations sur les connexions à vos sites et les erreurs survenues lors de la tentative de récupération des fichiers manquants. Vous pouvez utiliser ces fichiers de log à des fins de débogage de sites Web.

    Pour en savoir plus sur l'utilisation des logs d'accès au serveur Web, reportez-vous à la section Fichiers de log.

  • Accès au Catalogue d'applications. Vous pouvez voir et installer des applications sur les sites Web. Pour en savoir plus sur les applications, consultez la section Applications Web.
  • Accès à WordPress Toolkit. Gérez les instances WordPress installées sur vos sites Web à l'aide d'une interface de gestion unique. Pour en savoir plus, consultez la section WordPress Toolkit.
  • Gestion de la sécurité WordPress Toolkit. Gérez les options de sécurité disponibles dans WordPress Toolkit.
  • Configuration des options de scripts Web potentiellement dangereux allant à l'encontre de la politique du fournisseur Permet de passer outre la politique de sécurité de l'hébergement si elle est appliquée par le fournisseur.
  • Gestion des domaines. Configurez et gérez les nouveaux sites Web.

    Pour en savoir plus sur la configuration des sites Web, consultez les sections Configurer votre premier site Web et Ajouter des domaines.

  • Gestion des sous-domaines. Configurez et gérez de nouveaux sites Web avec des adresses comme forum.exemple.com.

    Pour en savoir plus sur la configuration de sous-domaines, consultez la section Ajouter des sous-domaines.

  • Gestion d'alias de domaine. Configurez et gérez des noms de domaine supplémentaires pour un site.

    Pour en savoir plus sur la configuration des alias de domaine, consultez le chapitre Ajouter un alias de domaine.

  • Gestion des comptes FTP supplémentaires. Créez et gérez les comptes FTP supplémentaires. Pour permettre la collaboration sur le contenu du site Web, vous pouvez configurer des comptes FTP pour d'autres utilisateurs et indiquer les répertoires du site qui doivent leur être accessibles.

    Pour en savoir plus sur la configuration des comptes FTP, consultez la section Ajouter un compte FTP.

  • Gestion des applications Java. Installez et gérez les applications Java distribuées dans les archives WAR et obtenues séparément de vendeurs tiers ou développeurs d'applications.
  • Changement des paramètres de messagerie. Gérez les paramètres du serveur de messagerie, comme le client de messagerie Web, l'activation ou la désactivation d'un service de messagerie pour un domaine.
  • Gestion des listes de diffusion. Configurez et gérez les listes de diffusion.

    Pour en savoir plus sur la configuration et l'utilisation des listes de diffusion, reportez-vous à la section Utiliser les listes de diffusion.

  • Gestion des paramètres des performances de l'hébergement. Limitez la bande passante et le nombre de connexions aux sites Web.

    Pour en savoir plus sur la restriction de l'utilisation de la bande passante de sites, consultez la section Limiter la bande passante et le nombre de connexions aux sites Web.

  • Gestion du pool d'applications IIS. Configurez les préférences personnalisées pour le pool d'applications IIS (disponible uniquement sur les comptes client sous Windows).

    Pour en savoir plus sur la configuration du pool d'applications IIS, reportez-vous à la section Configurer le pool d'applications IIS (Windows).

  • Gestion des autres droits d'écriture et de modification. Configurez d'autres droits d'écriture/de modification pour les sites Web utilisant des bases de données basées sur les fichiers (disponible uniquement sur les comptes client basés sur Windows).
  • Attribution d'un quota de disque dur. Adaptez les quotas de disque dur sur votre espace disque s'ils sont pris en charge par votre compte client.
  • Sélection du serveur de bases de données. Si plusieurs serveurs de bases de données sont disponibles, sélectionnez-en un pour créer des bases de données.
  • Accès à distance pour les utilisateurs de la base de données. Configurez les paramètres de contrôle d'accès à distance personnalisés pour les comptes utilisateur de bases de données. Pour en savoir plus, consultez la section Configurer des règles d'accès personnalisées.
  • Accès aux opérations avancées : copie du site Web. Copiez le contenu de l'un de vos sites Web vers un autre site Web ou un stockage FTP. Pour en savoir plus, consultez la section Utiliser un site provisoire.
  • Gestion des répertoires protégés par un mot de passe. Protégez un ou plusieurs répertoires de votre site Web avec des mots de passe. Les visiteurs devront saisir un mot de passe lorsqu'ils parcourent des répertoires protégés. Pour en savoir plus, consultez la section Restriction de l'accès au contenu.
  • Gestion des comptes utilisateur supplémentaires. Gérez des comptes utilisateur supplémentaires. Pour en savoir plus, consultez la section Gérer les comptes utilisateur supplémentaires.
  • Gestion des autres droits d'écriture et de modification (Windows). Accordez à votre utilisateur système les droits d'accès complets aux dossiers httpdocs et httpsdocs.
  • Gestion du Pool d'applications IIS (Windows). Activez/désactivez le pool d'applications dédié de votre site Web et définissez la part maximale du CPU que le pool peut utiliser. Pour en savoir plus, consultez la section Configurer le pool d'applications IIS.
