Créer des bases de données

Voir le tutoriel vidéo

Vous pouvez créer et supprimer des bases de données depuis Sites Web & Domaines > Base de données.

Vous pouvez créer une base de données en indiquant son nom. Si vous disposez des droits correspondants grâce à votre abonnement, vous pouvez choisir un type de base de données et un serveur de bases de données sur lequel la base de données doit être créée.

Vous pouvez sélectionner un site Web sur lequel la base de données sera utilisée via le champ Site associé. Cette option est pratique si vous avez de nombreux sites et bases de données, car vous verrez quelle base de données est utilisée par quel site directement depuis la page Bases de données.

Utilisateurs de bases de données

Au moins un compte utilisateur doit être associé à la base de données. Sinon, il est impossible d'y accéder. Vous pouvez créer un utilisateur de base de données pendant la création de bases de données. Pour cela, cochez la case Créer un utilisateur de bases de données. Vous pouvez également sélectionner un utilisateur par la suite. Par exemple, si vous voulez utiliser l'un des comptes utilisateur existants. Pour en savoir plus, consultez la section Gérer les comptes utilisateur de la base de données.

Pour sélectionner l'utilisateur de la base de données pour accéder à l'une de vos bases de données, cochez la case L'utilisateur dispose d'un accès à toutes les bases de données au sein de l'abonnement sélectionné.

Si vous avez les droits correspondants, vous pouvez définir les règles du contrôle d'accès pour un utilisateur de base de données. Pour en savoir plus, consultez la section Configurer des règles d'accès personnalisées.

Suppression d'une base de données

Pour supprimer une base de données, cliquez sur le lien Supprimer la base de données affiché pour cette base de données sous Sites Web & Domaines > Bases de données. Remarque : vous pouvez supprimer uniquement les bases de données et les utilisateurs de bases de données qui ne sont pas utilisés par les applications APS installées. Si le lien n'est pas disponible, cela signifie que la base de données est utilisée par une application et que vous ne pouvez la supprimer qu'en supprimant l'application concernée.

 

Accéder aux bases de données

Lorsque vous avez créé une base de données et affecté un utilisateur à cette base de données, vous pouvez y accéder depuis Plesk à l'aide du lien correspondant dans Sites Web & Domaines > Bases de données. Les liens sont différents selon le type de base de données :

  • phpMyAdmin pour les bases de données MySQL
  • pgMyAdmin pour les bases de données PostgreSQL
  • myLittleAdmin pour les bases de données SQL Server

Remarque : s'il n'y a aucun lien, contactez votre hébergeur.

Lorsque vous accédez à une base de données avec ces outils, Plesk utilise automatiquement les Identifiants d'accès de l'utilisateur de la base de données.

Si vous voulez accéder à votre base de données à l'aide d'autres outils (comme SQL Server Management Studio) ou si vous voulez autoriser une autre application à y accéder, vous devrez spécifier les paramètres suivants à ces outils et applications :

  • Pour vous connecter à MySQL et à PostgreSQL, vous devez indiquer le nom de l'hôte (ou l'adresse IP) ainsi que le nom d'utilisateur et le mot de passe de l'utilisateur de la base de données. Pour afficher cette information, cliquez sur le lien Infos de connexion sous le nom de la base de données.
  • Pour vous connecter à Microsoft SQL Server, vous devez également indiquer l'instance Microsoft SQL Server. Plesk utilise l'instance nommée MSSQLSERVER2008 pour Microsoft SQL Server 2008. C'est pourquoi vous pouvez indiquer l'hôte comme suit : mydomain.com\MSSQLSERVER2008 ou 123.123.123.123\MSSQLSERVER2008. Pour Microsoft SQL Server 2005, Plesk utilise l'instance par défaut (MSSQL 2005). Vous n'avez donc pas besoin de préciser le nom de l'instance.
