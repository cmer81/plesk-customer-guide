Paramètres généraux

Pour afficher les paramètres d'hébergement généraux d'un site Web, cliquez sur Paramètres d'hébergement sous le nom du site dans Sites Web & Domaines.

Les paramètres généraux d'hébergement d'un site Web sont divisés en plusieurs groupes :

Paramètres de base
  • Nom de domaine. Nom de domaine que vous enregistrez avec un registrar de domaine et utiliserez pour ce site Web.

    Remarque : selon la politique de votre hébergeur, vous pouvez ne pas être autorisé à modifier le nom du domaine principal de votre abonnement. Par défaut, le nom de domaine correspond au nom de l'abonnement.

  • Type d'hébergement. Le type d'hébergement (Site Web, Redirection et Aucun hébergement Web) définit le "comportement" du site Web. Par défaut, tous les sites Web appartiennent au type d'hébergement "Hébergement du site Web" car ils sont physiquement hébergés sur le serveur.

    Pour modifier le type d'hébergement, utilisez le lien Modifier. Pour en savoir plus sur les types d'hébergement, consultez la section Types d'hébergement.

    Pour suspendre le site Web et toutes les listes de diffusion et boîtes mail hébergées sous le nom du domaine du site Web, cliquez sur l'option Suspendre.

    Remarque : les autres paramètres de base dépendent du type d'hébergement sélectionné. Consultez la section Types d'hébergement.

  • Statut du site Web. Le statut du site Web définit l'accessibilité du site Web dans les navigateurs et depuis les services d'hébergement disponibles. Outre son fonctionnement normal, le site peut être suspendu : ainsi, il ne s'ouvre pas dans les navigateurs. De plus, les fonctionnalités d'hébergement sur le site (comme le service de messagerie et le service DNS) peuvent être désactivées. Si vous souhaitez désactiver temporairement votre site, par exemple, à des fins de maintenance, modifiez le statut. Consultez la section Statut du site Web.
  • Racine du document ou Document root. Ce paramètre s'affiche pour les domaines avec le type d'hébergement Site Web. Consultez la section Hébergement du site Web.
  • Domaine préféré. Ce paramètre s'affiche pour les domaines avec le type d'hébergement Site Web. Consultez la section Hébergement du site Web.
Paramètres de sécurité
  • Prise en charge SSL/TLS. Sélectionnez cette option pour protéger les transactions avec votre site via le protocole SSL/TLS qui chiffre toutes les données et les transfère sur une connexion sûre.
  • Redirection 301 permanente (SEO vérifié) de HTTP vers HTTPS. Si vous sélectionnez cette option, tous les visiteurs de votre site Web seront automatiquement redirigés de la version HTTP vers la version HTTPS de votre site avec le code d'état 301 HTTP (redirection permanente). Ce type de redirection est fiable niveau SEO car les moteurs de recherche indexent uniquement la version HTTPS du site et les capacités de classement dans les résultats de recherche ne seront pas perdues, contrairement au code d'état 302, avec redirection temporaire. Vous trouverez plus d'informations sur https://moz.com/learn/seo/redirection. Cette option n'est disponible que si la prise en charge SSL/TLS est activée. Lorsque vous utilisez cette option, vérifiez que vous disposez bien d'un certificat SSL/TLS valide car les navigateurs mettent en cache un site avec une redirection 301 permanente ; si un certificat auto-signé est utilisé, les visiteurs du site voient des messages d'avertissement à chaque visite du site.
  • Certificat. Pour utiliser SSL/TLS, installez un certificat SSL/TLS sur le site Web (voir la section Protéger les connexions avec des certificats SSL/TLS), puis sélectionnez l'un des certificats disponibles.

Remarque : les certificats SSL/TLS faisant partie du processus de chiffrement concernent généralement un seul nom de domaine sur une adresse IP unique ; tout site nécessitant une protection SSL/TLS doit donc être hébergé sur une adresse IP dédiée. Exception : les sous-domaines, que vous pouvez protéger avec un certificat Wildcard. Installer un certificat SSL/TLS sur un serveur Web hébergeant plusieurs sites Web avec différents noms de domaine sur une même adresse IP est possible sur le plan technique mais fortement déconseillé : il y aura chiffrement mais les utilisateurs recevront des messages d'avertissement lorsqu'ils essaieront de se connecter au site sécurisé.

Statistiques et scripts Web
  • Langages de script. Précisez les langages de script et de programmation que vous souhaitez prendre en charge avec votre site. Si vous choisissez PHP, vous pouvez également sélectionner une version et un type de gestionnaire PHP personnalisés. Vous trouverez davantage de paramètres PHP dans l'onglet PHP. Pour en savoir plus sur les gestionnaires PHP et la configuration PHP, consultez les Paramètres PHP.
  • Statistiques Web. Sélectionnez le logiciel de statistiques Web que vous voulez utiliser pour voir les rapports et graphiques sur les visiteurs de sites Web. Si vous voulez protéger le répertoire contenant les fichiers de statistiques Web avec vos nom d'utilisateur et mot de passe FTP, sélectionnez l'option "Protéger l'accès à vos statistiques Web avec votre nom d'utilisateur et mot de passe FTP". Remarque : si vous ne sélectionnez pas cette option, tout internaute peut accéder à vos statistiques Web.
  • Documents d'erreurs personnalisés. Lorsque des internautes visitant un site demandent des pages que le serveur Web ne trouve pas, il génère et affiche une page HTML standard contenant un message d'erreur. Si vous voulez créer vos propres pages d'erreur et les utiliser sur le serveur Web, cochez la case Documents d'erreurs personnalisés.
  • Autres droits d'écriture et de modification (disponible uniquement pour l'hébergement Windows). Cette option est requise si les applications Web sur le site vont utiliser un fichier de base de données (comme Jet) se trouvant à la racine du répertoire httpdocs ou httpsdocs . Remarque : si vous choisissez cette option, la sécurité du site Web sera sérieusement compromise.
