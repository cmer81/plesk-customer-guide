Configurer les pages d'erreur personnalisées sur les serveurs Windows

Pour configurer le serveur Web pour afficher les pages d'erreur personnalisées pour un site ou un répertoire dans un site :

  1. Activez la prise en charge des documents d'erreur personnalisés.
    1. Allez dans Sites Web & Domaines et cliquez sur le bouton Paramètres d'hébergement à côté du nom de domaine dont vous avez besoin.
    2. Cochez la case Documents d'erreur personnalisés.
    3. Cliquez sur OK.
  2. Allez dans Sites Web & Domaines et repérez le nom de domaine dont vous avez besoin.
  3. Cliquez sur Répertoires virtuels pour voir la liste des documents d'erreur pour le répertoire Web root. Les documents d'erreur situés ici sont utilisés pour toutes les pages Web du site sélectionné. Si vous voulez personnaliser les pages d'erreur d'un répertoire virtuel spécifique donné, parcourez ce répertoire.
  4. Cliquez sur l'onglet Documents d'erreur, puis cliquez dans la liste sur le document d'erreurs concerné.
    • Pour utiliser le document par défaut fourni par IIS pour cette page d'erreur, sélectionnez Par défaut dans le menu Type.
    • Pour utiliser un document HTML personnalisé qui se trouve déjà dans le répertoire error_docs dans le répertoire hôte virtuel du domaine, sélectionnez Fichier dans le menu Type, puis entrez le nom du fichier dans le champ Emplacement.
    • Pour utiliser un document HTML personnalisé qui se trouve dans un autre répertoire que le répertoire error_docs d'un domaine, sélectionnez l'URL dans le menu Type, puis saisissez le chemin d'accès à votre document dans le champ Emplacement. Le chemin d'accès doit être constitué de "http://<votre nom de domaine>" et du chemin d'accès vers le root de l'hôte virtuel (soit <vhosts>\<domain>\httpdocs).

      Exemple : votre nom de domaine est exemple.com. Vous avez créé un fichier appelé forbidden_403_1.html et vous l'avez enregistré dans le répertoire httpdocs\my_errors. Pour utiliser ce fichier en tant que document d'erreur, vous devez saisir l'URL dans le champ Emplacement : http://example.com/my_errors/forbidden_403_1.html.

      Remarque : pour des raisons de sécurité, vous ne pouvez pas utiliser de connexion via FTP ou via Gestionnaire de fichiers dans Plesk pour envoyer votre document d'erreurs personnalisé sur le serveur. Utilisez une connexion RDP si ce droit vous a été accordé. Sinon, contactez votre hébergeur pour obtenir de l'aide.

  5. Après s'être réinitialisé, le serveur Web utilisera désormais vos documents d'erreur personnalisés.
