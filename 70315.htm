Fonctions d'hébergement disponibles pour vos sites Web

En fonction de votre pack de services, les fonctions d'hébergement suivantes peuvent être disponibles pour vos sites Web (listées dans Compte > onglet Options d'hébergement) :

  • Prise en charge SSL/TLS. Permet de sécuriser les connexions aux sites Web avec le chiffrement SSL/TLS.

    Pour en savoir plus sur la sécurité des sites avec SSL/TLS, reportez-vous à la section Sécuriser les connexions avec les certificats SSL/TLS.

  • Statistiques Web. Vous permet d'afficher les statistiques des visites de sites Web dans les graphiques et diagrammes.

    Pour en savoir plus sur l'affichage des statistiques de visite des sites Web, consultez la section Voir les statistiques.

  • Documents d'erreurs personnalisés. Permet de créer des pages HTML personnalisées et de configurer le serveur Web pour les voir au lieu d'afficher des messages d'erreur classiques de type 404 Not Found.

    Pour en savoir plus sur la configuration des documents d'erreurs personnalisés, consultez la section Configurer les documents d'erreurs personnalisés.

  • Prise en charge des langues de script et de programmation comme PHP, CGI, Perl, Python, Microsoft ASP, ASP.NET, SSI.
  • Pool d'applications IIS dédié (Windows). Cette option permet l'isolement et une meilleure stabilité des applications Web fonctionnant sur les sites.
  • Autres droits d'écriture et de modification (Windows). Cette option permet d'utiliser une base de données basée sur un fichier (comme Jet) située à la racine du dossier httpdocs ou httpsdocs.
  • Autoriser les utilisateurs Web à utiliser les scripts. Permet le script sur les pages Web disponibles via des URL de type http://exemple.com/~<nom d'utilisateur>/<page Web>, où <nom d'utilisateur> fait référence à un utilisateur Web. Les utilisateurs Web sont des individus qui n'ont pas besoin d'avoir leur propre nom de domaine. Ce service est très populaire dans les établissements scolaires qui hébergent les pages personnelles de leurs étudiants et de leur personnel.
