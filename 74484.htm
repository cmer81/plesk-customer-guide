Suivi des demandes ayant échoué (Windows)

La fonction Suivi des demandes ayant échoué vous aide à résoudre les problèmes avec vos sites Web : elle rend les fichiers de demandes IIS ayant échoué accessibles dans l'interface de Plesk. Les fichiers des demandes ayant échoué contiennent des informations comme les URL demandées, le nom du pool d'application pertinent, le code d'état HTTP, etc.

Pour afficher la liste actuelle des fichiers de demandes ayant échoué, allez sous Sites Web & Domaines > Suivi des demandes ayant échoué. Vous pouvez cliquer sur tous les fichiers de demandes ayant échoué pour afficher leur contenu. Vous pouvez aussi télécharger des fichiers dans une archive ZIP ou supprimer les fichiers devenus inutiles. Vous pouvez lancer un nouveau suivi en indiquant des codes d'état HTTP ainsi que les composants HTTPS pour lesquels vous voulez obtenir des fichiers de demandes ayant échoué. Une fois que le suivi a commencé, vous pouvez modifier ces paramètres à tout moment.
